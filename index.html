My Portfolio Website

My Portfolio Website is a comprehensive platform designed to showcase my professional experience, projects, and skills. Built with user experience at its core, the site features an engaging hero section with animation, an experience section detailing my work history, a projects section with detailed project pages, and an about page providing personal insights. The site goes beyond basic browsing by incorporating interactive elements that offer visitors a dynamic way to explore my work and background.

An easy-to-navigate interface allows users to seamlessly transition between sections. The contact section links directly to my LinkedIn, GitHub, and email, facilitating effortless communication. A prominently placed resume button enables visitors to view or download my resume in PDF form with a single click. Custom UI/UX designs were created in Figma to ensure a cohesive and intuitive browsing experience across all devices.

Technologies and Tools:

  • Front-End: JavaScript, HTML, CSS
  • Development & Collaboration: GitHub, Visual Studio Code
  • Design & Prototyping: Figma, Adobe (Photoshop, Illustrator)

Development Process & Methodologies

Agile & Scrum Methodologies

The development process followed Agile principles, breaking down the project into manageable sprints. Scrum practices were employed to track progress, review completed work, and adapt to changes. This approach ensured that key features like the animated hero section, interactive project gallery, and responsive design were prioritized and delivered efficiently.

Testing

1. Unit Testing: Ensured individual components and scripts functioned correctly.
2. Manual Testing: Verified that animations, navigation, and interactive elements worked seamlessly.
3. Responsive Testing: Confirmed that the website was fully responsive and provided a consistent experience across various devices and screen sizes.

Figma Design Process (User Stories, Site Map, Wireframes, Prototype)

User Stories

User stories were developed to prioritize key user experiences, focusing on visitors, recruiters, and fellow developers.

Site Map

An organized site map was created to outline the website's structure, including key sections like the Home Page with a hero animation, navigation bar, and resume button. It features an Experience section with detailed professional history, a Projects gallery with search and filter functionality, individual project pages, an About page highlighting personal background, and a Contact section with links to LinkedIn, GitHub, and email. A footer contains social media icons and copyright information, ensuring smooth navigation and logical flow throughout the site.

Wireframes

Low-fidelity wireframes were created to visualize the layout and content arrangement across pages, focusing on key elements such as navigation menu placement, content section organization, and responsive design for different screen sizes. The goal was to ensure a user-friendly interface, allowing visitors to easily navigate and interact with the site's features.

Prototype (Desktop & Mobile)

High-fidelity prototypes were developed in Figma, featuring cohesive branding through a consistent color palette and typography. User testing was conducted to gather feedback on usability and aesthetics, leading to iterative improvements. This process ensured a polished, user-centric design that met the needs identified in the user stories.

Code Highlights

Typing Animation Effect

Code Snippet: index.js

Challenge: Create a dynamic typing animation for the hero section that types out text character by character, including a blinking cursor effect.

Solution:

  • Variables Initialization: Defined the text strings and target HTML elements.
  • Typing Function: Created a function to incrementally display text using setInterval.
  • Cursor Styling: Implemented a blinking cursor using CSS animations.
  • Sequencing: Managed timing to coordinate the typing of multiple text elements.

Reflection: Implementing this feature added a personal and interactive element to the landing page, capturing visitor interest immediately.

Projects Filtering and Sorting Functionality

Code Snippet: index.js

Challenge: Develop an intuitive search and filter system within the projects section to allow users to find projects based on various criteria.

Solution:

  • Data Structure: Stored project details in an array of objects, including tags and titles.
  • Search Functionality: Filtered projects based on user input matching titles or tags.
  • Sorting Mechanism: Provided options to sort projects by different attributes.
  • Dynamic Rendering: Updated the displayed projects based on the filtered and sorted data.

Reflection: This feature enhanced the user experience by making it easier to navigate and explore projects, demonstrating attention to usability and interactive design.

Smooth Scrolling Navigation

Code Snippet: index.js

Challenge: Implement smooth scrolling to anchor links without default browser behavior causing abrupt jumps or URL hash changes.

Solution:

  • Event Listeners: Attached click events to navigation links.
  • Prevent Default Behavior: Used event.preventDefault() to stop the default action.
  • Smooth Scrolling: Utilized scrollIntoView({ behavior: 'smooth' }) for a seamless transition.
  • History Management: Employed the History API to update the URL without reloading the page.

Reflection: This approach improved user experience by providing smooth transitions between sections, enhancing the site's professionalism and polish.
